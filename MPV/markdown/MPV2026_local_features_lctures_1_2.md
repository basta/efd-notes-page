## Design of Local Features

<!-- image -->

- 'Local Features' are regions, i.e. in principle arbitrary sets of pixels (not necessarily contiguous) with:
- ◼ High repeatability, (ideally invariance ) under
- Illumination changes
- Changes of viewpoint =&gt;  geometric transformations i.e. are distinguishable in an image regardless of
- viewpoint/illumination =&gt; are distinguished regions
- ◼ Are robust to occlusion =&gt; must be local
- ◼ Must have discriminative neighborhood =&gt; they are 'features'
- Methods based on local features/distinguished regions (DRs) formulate computer vision problems as matching of some representation derived from DR (as opposed to matching of entire images)

## Drop the descriptor completely? Possible, but

<!-- image -->

The main reason why we need a local feature descriptor is efficiency:

- ◼ Correspondences can be established by finding nearest neighbors by Euclidean (float) or Hamming (binary) distance.
- Which can be efficiently calculated via approximate NN libraries, e.g. FLANN, FAISS, etc.
- ◼ Can be avoided, e.g. the matching done by energy minimization, but more costly.
- ◼ See Optical flow (no descriptor, special constraints)

## SIFT properties

<!-- image -->

- ◼ Based on gradient orientations, which are robust to illumination changes
- ◼ Spatial binning gives tolerance to small shifts in location and scale, affine change.
- ◼ Explicit orientation normalization
- ◼ Photometric normalization by making the SIFT vector unit norm
- ◼ Orientation histogram gives robustness to small local deformations
- ◼ SIFT performance depends on a lot of implementation details

## SIFT Descriptor

<!-- image -->

- ◼ By far the most commonly used distinguished region descriptor:
- fast
- compact
- works for a broad class of scenes
- source code available in OpenCV, vlfeat, kornia, etc.
- Large number of ad hoc parameters. Enormous followup literature on both 'improvements' and
- improvements [RootSIFT, HoG, Daisy]
- GLOH, HoG:  different grid, not 4x4, not necessarily a square
- Daisy: many parameters optimized
- RootSIFT: square root normalization
- ◼ SIFT dominance currently challenged by the CNN-based descriptors
